### Key Technical Context
- **Tailwind must be via PostCSS** (Story 1.1 removes CDN) — shadcn/ui requires this, which is already done
- **cn() utility** uses `clsx` + `tailwind-merge` — standard shadcn/ui pattern at `src/lib/utils.ts`
- **Component install:** `npx shadcn-ui@latest add button form input label dialog alertdialog sheet toast progress card select textarea separator`
- **Tema:** CSS variables in `globals.css` following shadcn/ui convention (hsl format for light/dark)
- **Toaster:** Must wrap app in `<Toaster />` provider in App.tsx for toast notifications
- **This story is foundational** — Stories 1.6 (UX Quick Wins), 1.9 (Progress/Retry/Onboarding) depend on these components

### Testing
- Test files: `src/lib/__tests__/utils.test.ts`
- Framework: Vitest + RTL (from Story 1.2)
- cn() utility must be tested for class merging behavior (clsx + tailwind-merge)
- Migrated component must render correctly with shadcn/ui styling

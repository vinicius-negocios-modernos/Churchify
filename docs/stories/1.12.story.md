## Story
Como um usuario do Churchify, eu quero que a IA analise o conteudo real dos sermoes via transcricao do YouTube, para que o conteudo gerado reflita fielmente o que foi pregado ao inves de adivinhar pelo titulo.

## Objetivo

Implementar transcricao real de videos do YouTube (atualmente a IA "adivinha" pelo titulo) usando YouTube Data API + transcricao, reescrever prompts para usar o conteudo real do sermao, e adicionar fallback para videos sem legenda.

### Key Technical Context
- **SYS-013 e o debito mais impactante em qualidade de output** — a IA atualmente "adivinha" o conteudo pelo titulo do video, resultando em conteudo generico
- **Opcoes de transcricao (spike Task 1 deve avaliar):**
  1. YouTube Data API v3 captions endpoint — requer OAuth, legendas podem nao existir
  2. Biblioteca de terceiros (youtube-transcript, youtubei.js) — scraping, menos confiavel
  3. Audio extraction + speech-to-text — complexo, custoso, melhor qualidade
- **Fallback obrigatorio:** Videos sem legenda devem ter mensagem clara e usar titulo+descricao como contexto
- **YouTube API key** deve ficar server-side na Edge Function (AC6), mesma infra de Story 1.10
- **Campo transcript (TEXT)** adicionado a tabela episodes — pode ser grande (10K+ chars para sermoes longos)
- **Prompts reescritos** devem ter template: "Baseado na transcricao a seguir do sermao: {transcript}" quando disponivel, e fallback "Baseado no titulo e descricao: {title} - {description}" quando nao disponivel

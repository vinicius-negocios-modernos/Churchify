## Story
Como um usuario do Churchify, eu quero que formularios tenham validacao inline acessivel, que o upload via drag-and-drop funcione, e que botoes interativos tenham aria-labels descritivos, para que a experiencia de uso seja acessivel e intuitiva sem depender de alert().

## Objetivo

Resolver os quick wins de UX e acessibilidade que nao dependem de persistencia de dados: substituir alert() por inline validation, adicionar aria-labels em botoes interativos, tornar D&D upload funcional, validar tamanho de arquivo, e corrigir contraste de cores.

## Debitos Resolvidos

- UX-002: alert() para validacao de form (3h)
- UX-003: D&D upload visual mas nao-funcional (4h)
- UX-004: Sem validacao de tamanho de arquivo (1h)
- A11Y-001: Menu mobile sem aria-label (0.5h)
- A11Y-002: Fechar menu sem aria-label (0.5h)
- A11Y-003: Form validation via alert() nao acessivel (2h) — resolvido junto com UX-002
- A11Y-006: Copy buttons sem aria-labels descritivos (2h)
- A11Y-007: Color contrast text-gray-400 ratio 2.9:1 falha AA (2h)

## Acceptance Criteria

- [x] AC1: Zero chamadas a alert() em todo o codebase. Validacao de form usa inline errors com aria-invalid e aria-describedby
- [x] AC2: D&D upload area aceita arquivos via drag-and-drop E via click (file input fallback)
- [x] AC3: Validacao de tamanho de arquivo: rejeita arquivos > 10MB com mensagem amigavel

### Key Technical Context
- **alert() replacement:** Use shadcn/ui Form component with react-hook-form for inline validation (aria-invalid + aria-describedby auto-handled)
- **A11Y-006:** Copy buttons in ResultsDisplay.tsx lack descriptive aria-labels — add contextual labels like "Copiar post para Instagram", "Copiar legenda para YouTube"
- **D&D upload:** Current upload area has visual styling but no onDrop/onDragOver handlers — implement with file input fallback
- **File size validation:** Max 10MB, show user-friendly error via Toast component (from Story 1.5)
- **Color contrast fix:** All instances of `text-gray-400` must change to `text-gray-500` or darker (ratio >= 4.5:1 for WCAG AA)
- **Depends on Story 1.5** for shadcn/ui Form, Toast, Dialog components

## Story
Como um desenvolvedor do Churchify, eu quero habilitar TypeScript strict mode, adicionar Error Boundaries e resolver debitos de polish (toast, form persistence, meta tags), para que o codebase seja mais seguro, resiliente e profissional.

## Objetivo

Habilitar TypeScript strict mode e resolver todos os erros resultantes, adicionar React Error Boundaries, implementar input validation/sanitization, e resolver debitos de polish restantes (form persistence, toast feedback, placeholder pages, etc).

### Key Technical Context
- **TS Strict mode:** Enabling `strict: true` will surface null-check errors across the codebase. Expect 20-50+ errors. Focus on proper null guards, not `as` casts
- **Toast:** Requires shadcn/ui Toast component (from Story 1.5). Use `useToast()` hook after generation completes
- **Loading skeletons:** Requires shadcn/ui Skeleton component (from Story 1.5/1.9)
- **Form persistence:** Use `sessionStorage.setItem/getItem` in SermonForm — save on change, restore on mount
- **Copy fallback:** `navigator.clipboard.writeText()` with `document.execCommand('copy')` fallback for older browsers

### Testing
- Test file location: `src/lib/__tests__/ErrorBoundary.test.tsx`, `src/lib/__tests__/validation.test.ts`
- Test standards: Vitest + RTL. Error Boundary tests: render child that throws -> verify fallback renders
- URL validation: test youtube.com, youtu.be, invalid URLs, edge cases

## Story
Como um usuario do Churchify, eu quero ver dados reais no dashboard com estatisticas dos meus episodios, para que eu tenha visibilidade do meu uso da plataforma ao inves de dados ficticios.

## Objetivo

Substituir o dashboard 100% hardcoded por dados reais do banco de dados, implementar empty state educativo para novos usuarios, e mostrar estatisticas reais (total de episodios, ultimo episodio, uso de API).

## Debitos Resolvidos

- SYS-005: Dashboard 100% hardcoded mock data (8h)
- SYS-016: Sidebar mostra dados hardcoded de igreja/plano (referenciado em AC7 — Task 6)
- UX-001: Dashboard empty state + dados reais (10h) — parcialmente sobreposto com SYS-005
- DB-006: Dashboard usa mock data (6h) — deduplicado

## Acceptance Criteria

- [ ] AC1: Dashboard mostra dados reais do banco: lista de episodios recentes, contagem total, ultimo episodio
- [ ] AC2: Quando usuario nao tem episodios, empty state educativo aparece: "Crie seu primeiro episodio" com CTA
- [ ] AC3: Cards de estatisticas mostram dados reais: total episodios, episodios este mes, ultimo processamento
- [ ] AC4: Loading skeleton aparece enquanto dados carregam (nao tela em branco)
- [ ] AC5: Dados atualizam automaticamente apos criar novo episodio (sem refresh manual)
- [ ] AC6: Zero dados hardcoded ou mock no dashboard em producao
- [ ] AC7: Igreja/plano no sidebar mostra dados reais do perfil do usuario

### Key Technical Context
- **Dashboard.tsx e 100% mock** — dados estaticos hardcoded, nenhuma query ao banco
- **SYS-016 (AC7):** Sidebar mostra nome da igreja e plano hardcoded — Task 6 busca esses dados do perfil do usuario no banco (tabela profiles ou churches, conforme schema de Story 1.7)
- **Empty state** e critico para retencao — novos usuarios veem tela vazia sem orientacao
- **useDashboardData hook** deve usar Supabase client: `supabase.from('episodes').select(...)` com filtro por user_id
- **Auto-refresh:** Apos criar episodio em NewEpisode, invalidar cache do dashboard (React Query invalidateQueries ou refetch manual)
- **Loading skeleton:** Usar shadcn/ui Skeleton component (dep Story 1.5)
- **Dep Story 1.7:** Schema de episodes e profiles deve existir para queries reais

### Testing
- **Framework:** Vitest + React Testing Library (configurados em Story 1.2)
- **Localizacao de testes:** `src/__tests__/` ou co-located `*.test.tsx`
- **Padrao:** Mock Supabase client para hooks; testar empty state, loading state, e data state separadamente

## Story
Como um usuario do Churchify, eu quero ver progresso visual durante processamento AI, poder tentar novamente em caso de erro, e receber orientacao no primeiro uso, para que eu nao abandone a plataforma durante esperas longas ou falhas inesperadas.

## Objetivo

Melhorar a experiencia do usuario durante processamento AI (progress stepper), em caso de falha (retry button), e no primeiro uso (onboarding flow). Estas melhorias sao criticas para retencao — sem elas, usuarios abandonam durante o processamento de 10-60s.

### Key Technical Context
- **Processamento AI leva 10-60s** — sem feedback visual, usuarios abandonam
- **shadcn/ui Progress** (dep Story 1.5) fornece componente de barra de progresso acessivel
- **shadcn/ui Dialog** (dep Story 1.5) fornece modal acessivel com focus trap para onboarding
- **aria-live="polite"** deve envolver a regiao do stepper para screen readers anunciarem mudancas
- **Retry deve preservar state** — usar state management do React (useState no form parent) para nao perder dados digitados
- **Onboarding flag:** Pode ser `localStorage.getItem('onboarding_completed')` inicialmente; migrara para profile no banco quando Story 1.7 estiver pronta
- **Nao ha toast/notification system atualmente** — esta story pode usar shadcn/ui Toast do Story 1.5

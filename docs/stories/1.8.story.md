## Story
Como um administrador de igreja no Churchify, eu quero que Row Level Security garanta que cada usuario so acesse dados da sua propria igreja, e que imagens sejam armazenadas no Supabase Storage com URLs seguras, para que haja isolamento de dados e performance adequada.

## Objetivo

Implementar Row Level Security (RLS) em todas as tabelas para garantir isolamento de dados entre usuarios/igrejas, configurar Supabase Storage buckets com policies de acesso, e migrar imagens de base64 em React state para URLs no Storage.

## Debitos Resolvidos

- DB-004: No RLS / security policies (4h)
- DB-005: No storage security policies (2h)
- DB-007: Images base64 em memoria apenas (parcial — 4h, complementado em Story 1.15)

### Key Technical Context
- **RLS policies depend on church_members table** created in Story 1.7 Task 12 (church_id UUID FK, user_id UUID FK, role TEXT, joined_at TIMESTAMPTZ, PK(church_id, user_id)). Policies use `EXISTS (SELECT 1 FROM church_members WHERE church_id = target.church_id AND user_id = auth.uid())` pattern
- **RLS must be enabled on ALL tables:** profiles, churches, episodes, church_members
- **profiles policy:** SELECT/UPDATE WHERE id = auth.uid()
- **churches policy:** SELECT/UPDATE WHERE EXISTS in church_members for user
- **episodes policy:** SELECT/INSERT/UPDATE/DELETE WHERE episode's church_id matches user's membership
- **Storage bucket:** `episode-images` with policies scoped to church membership
- **Base64 migration:** Current images stored as base64 in React state — migrate to Storage URLs (partial fix, completed in Story 1.15)
- **DB-007 note:** Images as base64 causes memory pressure — Storage URLs resolve this

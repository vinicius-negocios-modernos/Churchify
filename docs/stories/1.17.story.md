## Story
Como um desenvolvedor do Churchify, eu quero cobertura de testes >= 80% com testes E2E nos 4 fluxos criticos, para que tenhamos confianca na estabilidade da aplicacao e possamos detectar regressoes automaticamente.

Expandir cobertura de testes unitarios e de integracao para 80%, implementar testes E2E com Playwright para os 4 fluxos criticos, e realizar audit de performance com Lighthouse. Resolver debitos P3 restantes.

### Key Technical Context
- **Playwright:** Use `@playwright/test`. Configure 2 projects: `chromium` (desktop) and `mobile-chrome` (mobile viewport)
- **Test infrastructure:** Story 1.2 sets up Vitest + RTL. This story adds Playwright for E2E and expands unit/integration coverage
- **CI integration:** Add Playwright job to existing `.github/workflows/ci.yml` (from Story 1.13). Use `actions/cache` for Playwright browsers
- **Coverage tool:** Vitest built-in coverage with `@vitest/coverage-v8`. Target: 80% line coverage
- **Bundle analysis:** Use `rollup-plugin-visualizer` or `source-map-explorer` to analyze Vite build output

> **E2E Auth**: Para testes E2E com Google OAuth, usar estrategia de mock auth (bypass OAuth em ambiente de teste) ou conta de teste dedicada. Definir abordagem antes de Task 3 (E2E login). Opcoes: (1) Supabase test user com email/password, (2) storageState com session pre-autenticada, (3) mock auth provider em test env.

### Testing
- Test file location: `e2e/*.spec.ts` (Playwright), `src/**/__tests__/*.test.ts` (Vitest)
- Test standards: Playwright uses page object pattern. Vitest uses RTL best practices (query by role/label)
- Coverage: Run `npm run test:coverage` and verify >= 80% before marking complete
